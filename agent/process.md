# Process (TDD)

## Checklist (Mandatory)

- Read the three agent docs (architecture, process, roadmap)
- RED: write or update tests first
- RED: run tests and confirm failure
- GREEN: implement minimal change
- GREEN: run tests and confirm pass
- REFACTOR: improve code without changing behavior
- Update roadmap.md with progress
- Commit with a clear message

## Rules

- No code changes without tests
- Keep business logic in services, not controllers
- All tests must pass before commit
